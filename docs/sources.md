# Источники и приоритет

`envspec` собирает значения из нескольких источников и сливает их в детерминированном
порядке. Где какое значение взялось — всегда видно в `envspec doctor` (колонка origin).

## Источники

| Слой | Origin | Когда подключается | Зависимость |
|---|---|---|---|
| дефолты `Field(default=...)` | `default` | всегда | — |
| файл `.env` | `.env` | если файл существует (`dotenv_path`) | — |
| JSON-конфиг | `json` | если передан `json_path` | — |
| YAML-конфиг | `yaml` | если передан `yaml_path` | `envspec[yaml]` |
| переменные окружения | `environ` | если `use_environ=True` (по умолчанию) | — |
| оверрайды | `override` | если передан `overrides=...` | — |
| профиль | `profile` | если активен профиль с `overrides` | — |

Ядро — zero-deps. JSON читается стандартной библиотекой; YAML требует extra
`pip install "envspec[yaml]"` и при его отсутствии даёт понятный `SourceError`.

## Приоритет (от низшего к высшему)

```
default  <  .env  <  json  <  yaml  <  environ  <  override  <  profile
```

Каждый следующий слой перекрывает предыдущий для совпадающих ключей. То есть:

- переменная окружения перекрывает значение из `.env`;
- `.env` перекрывает дефолт поля;
- оверрайды (аргумент `overrides=`) перекрывают окружение;
- активный профиль имеет наивысший приоритет (форсит значения в `prod` и т.п.).

Порядок реализован в `sources.layered()` и зафиксирован тестами — он стабилен и
является частью публичного контракта (см. SPEC §2).

## Форматы файлов

**`.env`** — строки `KEY=value`, поддерживаются `#`-комментарии, `export KEY=...`
и кавычки вокруг значения:

```dotenv
# комментарий
export API_URL="https://api.example.com"
TIMEOUT_S=30
```

**JSON / YAML** — отображение верхнего уровня `ключ → значение`. Значения приводятся
к строкам (как из окружения), `null` → пустая строка; дальше работает обычное
приведение типов из `Field`. Не-объект верхнего уровня → `SourceError`.

```json
{ "API_URL": "https://api.example.com", "TIMEOUT_S": 30 }
```

## Пример: слияние и происхождение

```python
result = AppConfig.validate(
    dotenv_path=".env",      # .env: API_URL=from-dotenv
    json_path="conf.json",   # json: TIMEOUT_S=10
    overrides={"API_URL": "from-override"},
)
# environ: TIMEOUT_S=20

# Итог:
#   API_URL   = "from-override"  (origin: override — перекрыл .env)
#   TIMEOUT_S = 20               (origin: environ — перекрыл json)
```

`envspec doctor` покажет для каждой переменной её значение, origin и статус
(`ok` / `missing` / `error` / `deprecated`). Секреты (`secret=True`) везде маскируются
как `***`.

## Профили

Профиль — самый верхний слой. Он задаёт оверрайды дефолтов и может ужесточать
правила (например, в `prod` сделать поле обязательным). Активный профиль выбирается
аргументом `profile=` или переменной `ENVSPEC_PROFILE`. Подробнее — в README и SPEC §2.
